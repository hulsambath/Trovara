# /audit-architecture

Dispatch the `architecture-auditor` subagent against the whole `lib/` tree, then save its output to `docs/superpowers/audits/YYYY-MM-DD/04_architecture.md`.

## Steps

1. Resolve today's date and ensure the folder exists:
   ```bash
   mkdir -p "docs/superpowers/audits/$(date +%Y-%m-%d)"
   ```

2. Dispatch `architecture-auditor` with prompt:

   > Scan the whole `lib/` tree for MVVM violations, ServiceLocator bypasses, file-size violations, and style/i18n drift per your standing instructions. Return only the markdown table.

3. Write the returned table to `docs/superpowers/audits/<today>/04_architecture.md`, prefaced with:

   ```markdown
   # Architecture Audit — <today>

   Generated by `/audit-architecture`. Complements the `style-reviewer` subagent (feature-scoped) — this view is whole-codebase.

   <table from subagent>
   ```

4. Print counts by severity to stdout.

## Notes

- Idempotent — overwrites `04_architecture.md` for the current day.
- Run before opening any PR. For per-feature checks, prefer the `style-reviewer` subagent on the changed files.
