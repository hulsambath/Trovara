# /audit-security

Dispatch the `security-auditor` subagent against the whole `lib/` tree, then save its output to `docs/superpowers/audits/YYYY-MM-DD/01_security.md`.

## Steps

1. Determine today's date in ISO format:
   ```bash
   date +%Y-%m-%d
   ```

2. Ensure the audit folder exists:
   ```bash
   mkdir -p "docs/superpowers/audits/$(date +%Y-%m-%d)"
   ```

3. Dispatch `security-auditor` using the Agent tool with `subagent_type: "security-auditor"`. Prompt:

   > Audit the Trovara codebase for security issues per your standing instructions. Scope: entire `lib/` tree. Return only the markdown table.

4. Write the returned table to `docs/superpowers/audits/<today>/01_security.md`, prefaced with:

   ```markdown
   # Security Audit — <today>

   Generated by `/audit-security`. See `00_report.md` for aggregated cross-dimension view.

   <table from subagent>
   ```

5. Print the count of findings by severity to stdout for the user, e.g.:
   ```
   Security audit saved to docs/superpowers/audits/2026-05-22/01_security.md
   Findings: 2 Critical · 5 High · 3 Medium · 1 Low
   ```

## Notes

- This command is idempotent — running it twice on the same day overwrites `01_security.md`.
- Does not modify any other file in the audit folder.
- For the aggregated report, run `/audit` (orchestrator).
